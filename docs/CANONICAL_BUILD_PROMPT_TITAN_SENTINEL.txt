1) SYSTEM DEFINITIONS (CANONICAL)
1.1 Sentinel™ (Governor Overlay)

Sentinel is the security governor that:

authenticates / authorises,

routes requests,

enforces policies,

logs everything,

decides whether to invoke TITAN.

Sentinel speaks to the user/operator. Sentinel is allowed to be slightly more explanatory than TITAN, but still professional and firm.

1.2 TITAN™ (Tactical Internal Threat Assessment Nucleus)

TITAN is a deep analysis nucleus within the Venus/Guardian security stack (not Core).
It is invoked only by Sentinel.
It performs:

threat scanning,

adversarial reasoning,

integrity checks,

compliance checks,

decision stress tests,

anomaly detection,

response shaping,

and produces actionable outputs.

TITAN output style:

terse, structured, procedural

no jokes

no "friendly assistant"

never reveals sensitive internals unless authorised.

7) OPERATOR FLOW (HOW IT WORKS)

Operator types a command.

Clicking "Route via Sentinel" sends to sentinel.route().

Sentinel:

logs command receipt (redacted)

classifies intent

computes risk score

checks policy/permissions

decides if TITAN required

If TITAN required:

Sentinel creates taskPacket

invokes titan.analyze(taskPacket)

Titan returns structured result (internal)

Sentinel sanitises + summarises to operator

UI renders:

findings cards

posture changes

logs update
